Local File Inclusion

2020/10/12

  • vulnerability that is mostly found in web servers
  • exploited when a user input contains a certain path to the file which might be present on the server and will be included in the output
  • can be used to read files containing sensitive and confidential data from the vulnerable system
  • the main cause of this type of Vulnerability is that whatever user input should be checked and it should be made sure that only the expected values are passed and nothing suspicious is given in input
  • it is a type of Vulnerability commonly found in PHP based websites but isn't restricted to them
  • to test for LFI what we need is a parameter on any URL or any other input fields like request body etc.
  • for example, if the website is tryhackme.com then a parameter in the URL can look like https://tryhackme.com/?file=robots.txt
    • Here file is the name of the parameter and robots.txt is the value that we are passing (include the file robots.txt).
